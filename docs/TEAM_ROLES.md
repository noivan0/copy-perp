## 팀장별 역할

### 🛠️ 개발팀장 (dev-lead)
**담당**: 코드 품질 + 배포

- 버그 발견 즉시 수정 → `python3 -m py_compile` → `git commit` → `git push`
- Render 자동 배포 확인 (push 후 2~3분)
- 기술 부채 지속 해소
- QA팀에서 올라온 버그 우선 처리

**현재 집중 항목:**
- CORS credentials 누수 수정 (커밋 `04f6cea`, Render 재배포 대기 중)
- `/health/detailed` rate limit 추가
- 응답시간 헤더 추가

**커밋 컨벤션:**
```
fix: [문제 설명]
feat: [기능 설명]
perf: [성능 개선]
chore: [설정/인프라]
```

### 🧪 QA팀장 (qa-lead)
**담당**: 테스트 + 검증

- 매 배포 후 전 엔드포인트 smoke test
- 악조건 테스트 (잘못된 입력, 경계값, 보안 공격)
- Rate limit 동작 확인
- 이슈 발견 → 개발팀장에게 즉시 전달

**테스트 기준 URL:**
- Backend: https://copy-perp.onrender.com
- Frontend: https://copy-perp-frontend.vercel.app

**테스트 사이클:**
```
배포 감지 → smoke test → 이슈 문서화 → dev-lead 전달 → 재검증
```

## 🔄 협업 프로토콜

### 이슈 발견 시
```
발견팀 → 대장에게 보고 (형식 아래 참조)
대장 → 개발팀장에게 fix 지시
개발팀장 → fix + push
QA팀장 → 재검증
대장 → 노이반님 보고 (중요 이슈만)
```

### 이슈 보고 형식
```
[팀명] BUG/IMPROVE
심각도: CRITICAL/HIGH/MEDIUM/LOW
파일: (해당 파일)
문제: (한 줄 설명)
재현: (재현 방법)
```

### 자율 개선 사이클 (무한 반복)
```
1. 현재 상태 점검
2. 이슈 발견
3. 우선순위 결정 (전략팀)
4. 코드 수정 (개발팀)
5. 검증 (QA팀)
6. 배포 확인
7. → 1번으로
```
